# üî∑ Servlet Request (GET vs POST)

A request is generated when a user communicates with server via browser.

HTTP requests are categorized into different types, mostly:

- GET
    
- POST
    

---

# üî∑ POST Request

- Used to send data from client to server.
    
- Can transmit various types of data:
    
    - text
        
    - audio
        
    - video
        
    - image
        

### Features:

- No size limit (large data allowed)
    
- Data is secure (sent in body of HTTP request)
    

### HTML Example:

<form action="url" method="POST">

### Servlet Method:

protected void doPost(HttpServletRequest request,  
                      HttpServletResponse response)  
        throws ServletException, IOException

---

# üî∑ GET Request

- Used to request data from server.
    
- Can only transmit text data.
    

### Features:

- Limited size (4KB ‚Äì 8KB)
    
- Data is not secure (visible in URL)
    

### Generated By:

1Ô∏è‚É£ Using form without specifying method  
2Ô∏è‚É£ Using hyperlink  
3Ô∏è‚É£ Typing URL directly in browser

### Servlet Method:

protected void doGet(HttpServletRequest request,  
                     HttpServletResponse response)  
        throws ServletException, IOException

# üî∑ Cookies

- Small piece of information stored between client and server.
    
- Always travels with request.
    
- Used to track users.
    

### Working:

- Server creates cookie.
    
- Sends to client browser.
    
- Browser stores cookie.
    
- Cookie sent back with every request.
    

---

## Types of Cookies

1Ô∏è‚É£ Persistent Cookie

- Stored in browser until expiry time.
    
- Continues across browser sessions.
    

2Ô∏è‚É£ Non-Persistent Cookie (Session Cookie)

- Temporary.
    
- Deleted when browser closes.
    

---
